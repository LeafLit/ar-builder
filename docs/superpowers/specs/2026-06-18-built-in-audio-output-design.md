# 内置音效输出设计

## 目标

为 AR Builder 增加“音效”输出类型。用户可以在编辑页为每个识别状态选择一个内置音效，进入测试页后，当识别结果进入该状态时播放一次音效。

这补齐 PRD 中“音频效果”的第一版能力，也让原型更接近论文中“识别状态触发反馈”的交互表达。

## 范围

本次包含：

- 编辑页输出类型新增 `音效`。
- 提供 3 个内置音效选项：`提示音`、`成功音`、`警告音`。
- 保存后生成 `audio` 素材和 `playAudio` 绑定。
- 测试页在进入某个音效状态时播放一次。
- 同一状态持续识别时不重复播放。
- 识别丢失后再次进入同一状态时可以再次播放。
- 手动模拟识别和自动相机识别使用同一套播放规则。
- 如果浏览器阻止播放或设备不支持 Web Audio，页面不崩溃，并显示简短提示。

本次不包含：

- 上传自定义音频文件。
- 音量、循环、淡入淡出或多音轨混音。
- 计数达到阈值后再播放。
- 把音效和视觉输出同时绑定到同一状态。
- 项目级音效资源管理器。

## 用户体验

编辑页中，每个状态的“输出类型”下拉框新增 `音效`。

选择 `音效` 后，显示一个音效下拉框：

- `提示音`
- `成功音`
- `警告音`

音效输出不显示屏幕锚点控制，因为它没有位置和大小。

保存后进入测试页：

- 点击 `模拟识别状态 A`，如果状态 A 绑定了音效，则播放一次。
- 再次点击 `模拟识别状态 A`，因为状态没有变化，不重复播放。
- 点击 `模拟识别状态 B`，如果状态 B 绑定了音效，则播放状态 B 的音效。
- 自动识别确认某状态时，同样只在“进入状态”的瞬间播放一次。
- 自动识别丢失该状态后，再次确认同一状态时，可以再次播放。

测试页保留现有状态计数、识别灵敏度和 AR 输出区域。音效本身不占用 AR 画面；如果当前状态只有音效输出，AR 画面不显示虚拟物体。

## 数据模型

在 `projectTypes.ts` 中增加内置音效 ID：

```ts
export type BuiltInAudioId = "beep" | "success" | "alert";
```

新增音效草稿类型：

```ts
export type AudioOutputDraft = {
  assetType: "audio";
  audioId: BuiltInAudioId;
  name: string;
  transform: Transform;
};
```

`Asset` 增加可选字段：

```ts
audioId?: BuiltInAudioId;
```

保存音效输出时：

```ts
{
  id: "asset_audio_state_a",
  type: "audio",
  name: "提示音",
  audioId: "beep"
}
```

绑定使用已有 `playAudio` 动作：

```ts
{
  id: "binding_state_a",
  stateId: "state_a",
  action: {
    type: "playAudio",
    assetId: "asset_audio_state_a"
  }
}
```

## 架构

新增一个小的内置音效模块：

```text
src/features/ar/audioCatalog.ts
```

职责：

- 定义内置音效选项。
- 根据未知 ID 返回默认音效。
- 使用 Web Audio API 播放短音效。

编辑模块只负责选择音效并保存草稿。App 状态 reducer 负责把草稿转换成 `audio` 素材和 `playAudio` 绑定。测试模块负责在识别状态进入时调用播放器。

机器学习模块不感知音频逻辑；它仍然只输出“当前识别到了哪个状态”。

## 播放规则

测试页需要记录上一次已播放音效的状态：

- 如果当前确认状态为空：清空上一次音效状态。
- 如果当前确认状态等于上一次音效状态：不播放。
- 如果当前确认状态不同，并且该状态绑定的是 `playAudio`：播放一次并记录该状态。
- 如果当前确认状态不同，但绑定的是视觉输出：不播放音效，但仍记录当前状态，避免同一状态持续时重复检查。

这条规则与状态计数器类似，但保持独立，避免把计数逻辑和音频副作用绑死。

## 错误处理

浏览器可能因为自动播放策略或 Web Audio 支持问题阻止播放。默认播放器需要捕获错误并向测试页返回失败结果。

测试页显示简短提示：

```text
音效播放被浏览器阻止，请点一下页面后重试。
```

错误提示不影响识别、计数和视觉 AR 输出。

## 测试计划

自动化测试覆盖：

- 内置音效目录能返回默认音效。
- 编辑页可以保存音效输出。
- 编辑页能加载已有音效绑定继续编辑。
- App reducer 把音效草稿保存成 `audio` 素材和 `playAudio` 绑定。
- App reducer 保存视觉输出时会替换同状态旧音频素材和绑定。
- 测试页进入音效状态时调用播放器一次。
- 测试页同一状态持续识别时不重复播放。
- 测试页识别丢失后再次识别同一状态时再次播放。
- 音效播放失败时显示提示且页面不崩溃。

## 验收标准

- 用户可以在编辑页为状态 A / 状态 B 选择音效输出。
- 保存项目后重新打开，音效选择仍然存在。
- 手动模拟识别可以触发音效。
- 自动识别可以触发音效。
- 同一状态持续识别不会连续刷屏式播放。
- 所有现有文字、图片、3D 模型和状态计数功能保持可用。
