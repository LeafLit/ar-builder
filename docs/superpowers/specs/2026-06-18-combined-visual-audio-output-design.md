# 视觉和音效组合输出设计

## 目标

让同一个识别状态可以同时触发一个视觉输出和一个内置音效。例如：识别到拳头时，屏幕上显示 3D 小树，同时播放成功音。

这个功能延续前一步的内置音效输出，让音效不再只能作为单独输出类型使用，更适合做论文中“状态触发反馈”的 AR 原型。

## 范围

本次包含：

- 文字、图片、3D 模型输出可以选择一个“附加音效”。
- 附加音效选项包括：`不播放音效`、`提示音`、`成功音`、`警告音`。
- 保存后同一状态可以生成两个绑定：
  - `show`：显示视觉输出。
  - `playAudio`：进入状态时播放音效。
- 测试页识别到该状态时，同时显示视觉内容并播放音效。
- 同一状态持续识别时，视觉保持显示，音效不重复播放。
- 识别丢失后再次识别该状态时，音效可以再次播放。
- 保留纯音效输出类型：状态也可以只播放音效、不显示视觉内容。

本次不包含：

- 一个状态绑定多个视觉输出。
- 一个状态播放多个音效。
- 条件规则，例如计数达到 3 次才播放。
- 音量控制、循环播放或上传自定义音频。
- 完整可视化规则编辑器。

## 用户体验

编辑页中，用户先选择主输出类型：

- `文字`
- `图片`
- `3D 模型`
- `音效`

当主输出是 `文字`、`图片` 或 `3D 模型` 时，卡片底部出现一个“附加音效”下拉框：

- `不播放音效`
- `提示音`
- `成功音`
- `警告音`

当主输出是 `音效` 时，界面保持上一轮实现的样子，只显示“音效”选择，不显示屏幕锚点，也不再显示“附加音效”。

测试页行为：

- 状态 A 绑定文字 + 成功音：识别到状态 A 时显示文字，并播放成功音。
- 连续保持状态 A：文字继续显示，但成功音不重复播放。
- 切到状态 B 再切回状态 A：再次播放成功音。
- 状态 A 只绑定音效：不显示 AR 物体，只播放音效。

## 数据模型

视觉输出草稿增加可选音效：

```ts
type AudioCueDraft = {
  audioId: BuiltInAudioId;
  name: string;
};

type TextOutputDraft = {
  assetType?: "text";
  content: string;
  transform: Transform;
  audio?: AudioCueDraft;
};
```

图片和 3D 模型输出同样支持 `audio?: AudioCueDraft`。

纯音效输出继续使用：

```ts
type AudioOutputDraft = {
  assetType: "audio";
  audioId: BuiltInAudioId;
  name: string;
  transform: Transform;
};
```

保存视觉 + 音效时，项目内保存两个素材和两个绑定：

```ts
[
  {
    id: "asset_text_state_a",
    type: "text",
    name: "state_a 文字",
    content: "你好"
  },
  {
    id: "asset_audio_state_a",
    type: "audio",
    name: "成功音",
    audioId: "success"
  }
]
```

```ts
[
  {
    id: "binding_state_a",
    stateId: "state_a",
    action: {
      type: "show",
      assetId: "asset_text_state_a",
      transform,
      visible: true
    }
  },
  {
    id: "binding_audio_state_a",
    stateId: "state_a",
    action: {
      type: "playAudio",
      assetId: "asset_audio_state_a"
    }
  }
]
```

旧项目兼容：

- 只有 `binding_state_a` 的视觉输出继续正常加载。
- 只有 `binding_state_a` 的纯音效输出继续正常加载。
- 新项目如果有 `binding_audio_state_a`，编辑页会把它显示为“附加音效”。

## 架构

`appState.ts` 负责把一个状态草稿拆成一组素材和绑定。

`AuthoringScreen.tsx` 负责：

- 加载同一状态下的视觉绑定和音效绑定。
- 为视觉输出显示“附加音效”控制。
- 保存时把附加音效放进输出草稿。

`TestScreen.tsx` 负责：

- 独立解析同一状态下的 `show` 动作和 `playAudio` 动作。
- 视觉输出用现有 AR overlay 渲染。
- 音效输出沿用上一轮“进入状态播放一次”的规则。

`ruleEngine.ts` 当前只返回第一个匹配绑定。本次不扩展完整规则引擎，测试页和编辑页会用更直接的过滤逻辑查找 `show` 和 `playAudio`。后续如果做完整规则系统，再把这些查询收敛进规则引擎。

## 错误处理

- 附加音效素材缺失时，只显示视觉输出，不播放音效。
- 视觉素材缺失但音效存在时，页面显示视觉缺失提示，同时仍可以播放音效。
- 浏览器阻止音效播放时，继续显示上一轮的提示：`音效播放被浏览器阻止，请点一下页面后重试。`

## 测试计划

自动化测试覆盖：

- reducer 保存文字 + 音效时生成两个素材和两个绑定。
- reducer 从视觉 + 音效切回纯视觉时移除旧音效素材和绑定。
- 编辑页保存文字 + 附加音效。
- 编辑页加载已有视觉 + 音效绑定。
- 测试页同一状态同时显示视觉输出并调用播放器。
- 测试页同一状态持续时不重复播放音效。
- 纯音效输出继续不显示视觉物体。
- 现有文字、图片、3D、纯音效测试继续通过。

## 验收标准

- 用户可以给文字、图片、3D 模型添加一个附加音效。
- 保存项目后重新打开，附加音效仍然存在。
- 测试页可以同时显示视觉输出并播放音效。
- 纯音效输出仍然可用。
- 状态计数、识别灵敏度和 3D 旋转功能不受影响。
