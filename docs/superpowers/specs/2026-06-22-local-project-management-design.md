# 本机项目重命名和删除设计

## 背景

AR Builder 已经可以把当前原型保存到手机浏览器本地，并从首页继续编辑已保存项目。当前不足是：保存出来的项目都叫“AR Builder 本机项目”，用户无法改名，也无法删除试坏的项目。随着真机测试次数变多，本机项目列表会变得难以辨认。

PRD 的第一版项目管理范围已经包含“重命名项目”和“删除本地项目”。本轮补齐这两个能力。

## 目标

1. 用户可以在首页本机项目列表中重命名已保存项目。
2. 用户可以删除已保存项目。
3. 重命名后项目列表立即刷新，并显示新名称。
4. 删除后项目从列表移除，列表为空时显示“还没有保存的项目”。
5. 操作失败时给出简短中文状态提示。

## 非目标

1. 不做云同步、账号系统或跨设备删除。
2. 不做批量删除。
3. 不做撤销删除。
4. 不做导入/导出项目文件。
5. 不改变项目内部状态、素材、绑定、模型快照和设置结构。

## 用户体验

本机项目列表每个项目条目保留“继续编辑”按钮，并新增两个轻量操作：

1. “重命名”按钮：点击后在当前项目条目内显示名称输入框、“保存名称”和“取消”按钮。
2. “删除”按钮：点击后先显示一次确认态，按钮文案变为“确认删除”，旁边保留“取消”。用户再次确认才删除。

为了适合手机浏览器：

1. 不使用复杂弹窗，避免移动端键盘和弹窗叠在一起。
2. 输入框直接出现在项目卡片里，用户改完即可保存。
3. 空名称不保存，状态提示“项目名称不能为空。”。

## 数据设计

沿用现有 `ProjectRepository`：

```ts
type ProjectRepository = {
  list(): Promise<Project[]>;
  get(id: string): Promise<Project | undefined>;
  save(project: Project): Promise<void>;
  delete(id: string): Promise<void>;
};
```

不新增数据库表。重命名流程通过 `get(id)` 读取项目，再用 `save({ ...project, name, updatedAt })` 覆盖保存。删除流程调用已有 `delete(id)`。

`ProjectLibraryPanel` 只接收列表摘要，不能直接更新完整项目，因此新增两个回调：

```ts
onRenameProject?: (projectId: string, name: string) => void | Promise<void>;
onDeleteProject?: (projectId: string) => void | Promise<void>;
```

`App` 负责把这两个回调连接到项目仓库。

## 状态和错误处理

`ProjectLibraryPanel` 增加本地 UI 状态：

1. `renamingProjectId`：当前正在重命名的项目。
2. `draftProjectName`：当前输入框内容。
3. `deleteConfirmProjectId`：当前等待二次确认删除的项目。

状态提示规则：

1. 重命名成功：`项目名称已更新。`
2. 重命名为空：`项目名称不能为空。`
3. 重命名失败：`重命名项目失败，请稍后再试。`
4. 删除成功：`项目已删除。`
5. 删除失败：`删除项目失败，请稍后再试。`

如果项目在重命名时已经不存在，`App` 的回调直接返回，面板刷新后该项目会从列表消失。

## 兼容性

1. 旧项目仍然正常显示。
2. 不改变项目保存格式。
3. 不影响当前编辑页和测试页。
4. 不影响已经保存的识别模型、素材、绑定和识别灵敏度。

## 测试策略

1. `ProjectLibraryPanel`：点击“重命名”后显示输入框；输入新名称并保存会调用 `onRenameProject`，刷新列表并显示成功状态。
2. `ProjectLibraryPanel`：空名称不调用重命名回调，并显示错误提示。
3. `ProjectLibraryPanel`：点击“删除”后进入确认态；点击“确认删除”会调用 `onDeleteProject`，刷新列表。
4. `ProjectLibraryPanel`：删除失败时显示错误提示。
5. `App`：从首页重命名已保存项目后，仓库里的项目名称更新。
6. `App`：从首页删除已保存项目后，列表变为空。

## 风险与应对

1. 风险：误触删除。
   应对：删除需要二次确认。

2. 风险：项目名称为空导致列表难以辨认。
   应对：空名称不保存。

3. 风险：列表刷新与状态提示互相覆盖。
   应对：操作成功后先刷新列表，再设置明确的成功提示。

4. 风险：重命名影响当前打开项目。
   应对：本轮只在首页本机项目列表操作已保存项目；不改当前编辑中的 App 状态。
