## 2. 技术方向

- React + TypeScript + Vite + PWA。
- 手机浏览器优先，兼容鸿蒙、安卓、iOS。
- 真实摄像头识别。
- MobileNet embedding + 自训练状态分类器。
- 项目数据保存在当前浏览器本地 IndexedDB。
- 当前主路线是方案 A：相机背景 + 屏幕锚点 / 2D-3D 叠加，先保证跨手机浏览器可用。
- 方案 B：WebXR / 原生空间 AR 仍作为后续增强方向。

## 5. 当前边界和未完成内容

- 当前 MVP 仍固定为两个输入状态，但两个状态都可以改名。
- 采集样本已有基础管理能力：查看样本、删除坏样本、撤销删除、批量删除、补采样本和样本质量提示。后续还可以继续补充单张重拍和更细的自动质量检测。
- 识别准确率仍依赖现场光线、背景、样本质量和手机性能。
- 目前以屏幕锚点叠加为主，不承诺真实空间锚定。
- 项目导入 / 导出已有基础版本，可以用于备份、换设备测试和演示交接；后续可继续做格式版本升级、文件校验和更友好的冲突处理。
- 还没有登录、云同步和多设备同步。
- 还没有完整可视化规则编辑器或状态机编辑器。
- 还没有原生 App 封装、ARKit、ARCore 或华为 AR Engine 接入。

## 6. 建议下一步

优先级从高到低：

1. 继续打磨采集样本管理：单张重拍、删除前二次确认、更清楚的样本质量评分。
2. 增强项目导入 / 导出：格式版本提示、坏文件错误说明、跨设备导入后的训练状态说明。
3. 增加更多输出效果：粒子、更多简单动画或更多内置 3D 模型。
4. 支持从两个固定状态扩展到多个状态。
5. 优化状态切换体验：连续多帧确认参数、触发冷却时间和更细的过渡控制。
6. 再评估 WebXR / 原生空间 AR 方案。

## 2026-06-25 补充：二维码标记识别 Spike v1

- 测试页新增第三种“自动识别输入模式”：`二维码标记`。现在可选 `相机分类`、`颜色标记`、`二维码标记`。
- `相机分类` 仍然使用训练后的真实摄像头分类模型；`颜色标记` 仍然使用红 / 绿 / 蓝色块触发状态；`二维码标记` 使用摄像头读取二维码内容触发状态。
- 二维码 v1 规则：`ARBUILDER:1` 对应第 1 个状态，`ARBUILDER:2` 对应第 2 个状态，`ARBUILDER:3` 对应第 3 个状态。项目没有第 3 个状态时，`ARBUILDER:3` 不会触发。
- 为了方便高级排查，也支持 `state:状态ID` 这种直接写状态 ID 的二维码内容，但普通实机测试优先使用 `ARBUILDER:1 / 2 / 3`。
- 二维码识别依赖 `jsqr`，实现位置是 `src/features/testing/qrMarkerRecognizer.ts`，并已接入 `src/features/testing/TestScreen.tsx`。
- 本轮完成后需要实机测试：测试页能否切换到二维码标记、扫码触发第 1 / 2 / 3 个状态、无效二维码是否不乱触发、切回相机分类和颜色标记是否仍正常、AR 输出 / 计数 / 震动 / 音频是否仍跟随识别状态工作。
