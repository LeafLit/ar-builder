# 真机体验 A 方案实测清单

更新日期：2026-06-22

本文档用于第一版 PWA 真机验证。目标不是一次性覆盖所有原生能力，而是确认手机浏览器版本可以稳定完成：打开 App、安装到桌面、授权相机、采集样本、训练模型、编辑文字/图片/3D/音频输出、测试 AR 输出、本机保存项目并再次编辑。

默认真机测试地址：`https://leaflit.github.io/ar-builder/`

防缓存测试地址模板：`https://leaflit.github.io/ar-builder/?v=最新提交短哈希`

说明：如果刚刚合并或推送了新功能，手机浏览器和 PWA 可能还缓存旧版本。等 GitHub Actions 发布完成后，优先使用防缓存地址测试。如果代码还只是本地改动、没有推送到 `main`，线上测试地址暂时看不到这次新改动。

## 1. 验收目标

本轮通过后，App 应满足以下条件：

1. 手机可通过 HTTPS 地址打开 App。
2. 首页“真机体验检查”能显示安全连接、相机 API、PWA 安装、AR 模式。
3. 相机权限弹窗能正常出现，允许后可以看到后置摄像头画面。
4. 可以采集两个自定义命名状态的样本。
5. 可以在采集页查看样本缩略图，点击缩略图放大查看，删除拍坏样本，并继续补采样本。
6. 训练后可以进入编辑页，并保存文字、图片、3D 模型或音频输出。
7. 测试页可以启动自动识别；如果浏览器不支持空间 AR，则进入相机叠加降级体验。
8. 测试页可以调节识别灵敏度，并显示状态触发计数。
9. 本机项目可以继续编辑、重命名和删除。
10. Android、iOS、鸿蒙至少各完成一轮记录，明确哪些能力可用、哪些是降级。

## 2. 测试环境准备

### 2.1 部署要求

- 必须使用 HTTPS 域名访问，例如 `https://your-domain.com`。
- 当前 GitHub Pages 预期地址：`https://leaflit.github.io/ar-builder/`。
- 发布后建议使用防缓存地址测试，例如 `https://leaflit.github.io/ar-builder/?v=最新提交短哈希`。
- 本机开发调试可以使用 `localhost`，但手机访问电脑局域网 IP 时不能当作安全上下文。
- 构建命令：`pnpm.cmd run build`。
- 预览命令：`pnpm.cmd run preview`。

### 2.1.1 GitHub Pages 发布方式

- 每次推送到 `main` 后，GitHub Actions 会运行测试、构建并发布 `dist`。
- GitHub Pages 的发布源应设置为 GitHub Actions。
- GitHub Pages 构建会使用 `/ar-builder/` 子路径，确保 JS、CSS、manifest、图标和 service worker 都能在项目站点下加载。
- 每次完成需要真机测试的开发后，交付说明里都要列出需要测试的功能，并附上默认线上测试地址和防缓存测试地址模板。
- 默认在检查通过后自动提交并推送到当前远端分支，方便尽快真机测试；除非用户明确要求不要提交或不要推送。

### 2.2 手机准备

- 确认浏览器允许访问摄像头。
- 关闭低电量模式或后台限制，避免模型加载中断。
- 首次打开时保持网络稳定，因为 PWA 文件和模型相关资源会按需加载。
- 如果已经安装过旧版本 PWA，先从桌面删除旧图标，再重新添加。

## 3. 平台测试矩阵

| 平台 | 推荐浏览器 | 预期 AR 模式 | 重点观察 |
| --- | --- | --- | --- |
| Android | Chrome / Edge | 优先 WebXR，失败则相机叠加 | 是否显示“空间 AR（WebXR）”、安装提示、后置摄像头 |
| iOS | Safari | 相机叠加降级 | 添加到主屏幕后能否打开、摄像头权限是否保留 |
| 鸿蒙 | 系统浏览器 / Chrome 类浏览器 | 相机叠加降级为主 | getUserMedia 是否可用、PWA 是否可添加到桌面 |

## 4. 逐步测试流程

### 4.1 首页检查

1. 用手机打开 HTTPS 地址。
2. 查看首页“真机体验检查”。
3. 记录以下四项：
   - 安全连接
   - 相机 API
   - PWA 安装
   - AR 模式

通过标准：

- “安全连接”应显示“已满足”。
- “相机 API”应显示“浏览器支持”。
- Android 支持 WebXR 时，“AR 模式”显示“空间 AR（WebXR）”。
- iOS / 鸿蒙不支持空间 AR 时，允许显示“相机叠加降级”。

### 4.2 PWA 安装

Android：

1. 在浏览器菜单中选择“安装应用”或“添加到主屏幕”。
2. 从桌面图标重新打开 App。
3. 确认顶部浏览器地址栏是否消失或弱化。

iOS：

1. Safari 打开 App。
2. 点击分享按钮。
3. 选择“添加到主屏幕”。
4. 从桌面图标重新打开 App。

鸿蒙：

1. 使用系统浏览器或常用 Chrome 类浏览器打开 App。
2. 查找“添加到桌面 / 添加到主屏幕 / 安装应用”入口。
3. 从桌面图标重新打开 App。

通过标准：

- 桌面图标使用 AR Builder 图标。
- 从桌面打开后能进入首页。
- 首页检查面板能正常显示。

### 4.3 相机采集

1. 点击“新建项目”。
2. 点击“开启摄像头”。
3. 允许浏览器摄像头权限。
4. 分别选择两个状态，例如“拳头”和“巴掌”。
5. 修改状态名称，确认可以输入中文名称，也可以把输入框内容清空后重新输入。
6. 每个状态至少采集 1 张样本。
7. 查看“样本管理”区域，确认当前状态的样本缩略图能显示。
8. 点击一张缩略图，确认能打开大图预览；点击“关闭大图”后能回到采集页。
9. 删除一张拍坏的样本，确认该状态样本数量会减少。
10. 再点击“采集样本”补采一张，确认数量会重新增加。

通过标准：

- 可以看到摄像头画面。
- 拒绝权限时，页面要显示失败提示。
- 允许权限后，状态样本数量会增加。
- 状态名称可以正常编辑，删除首字或最后一个字时不会卡住。
- 样本管理区域能显示当前状态的样本缩略图和采集时间。
- 点击缩略图后，大图预览清晰可见，并且可以关闭。
- 删除坏样本后，该状态的样本数量会减少；补采后数量会增加。

### 4.4 训练与编辑

1. 进入训练页。
2. 确认页面显示“已满足真实训练条件”。
3. 点击“开始训练”。
4. 训练完成后进入编辑页。
5. 为两个状态分别尝试不同输出类型：文字、图片、3D 模型或音频。
6. 如果选择文字，填写一段文字；如果选择图片，上传一张手机相册里的图片。
7. 如果选择 3D 模型，切换内置模型，并调整旋转角度。
8. 如果选择音频，切换内置音效；如果选择视觉输出，也可以附加音频。
9. 调整每个视觉输出的“横向位置”“纵向位置”“大小”滑杆。
10. 点击“保存绑定”。

通过标准：

- 缺样本时训练页会列出缺少的状态。
- 样本齐全时训练页会显示满足条件。
- 保存后“下一步：测试”可点击。
- 再次进入编辑页时，已保存的文字、图片、3D 模型、音频、位置、大小和旋转仍能显示。

### 4.5 测试页识别

1. 进入测试页。
2. 点击“启动自动识别”。
3. 观察状态提示。
4. 对准刚才采集的真实物体或场景。
5. 调整“识别灵敏度”滑杆，确认当前相机识别结果会即时按新阈值变化。
6. 确认文字、图片或 3D 模型出现在编辑页设置的位置，大小和旋转符合设置。
7. 如果状态绑定了音频，确认识别到该状态时会播放对应音效。
8. 观察“状态计数”，确认状态真正切换时计数增加，并可点击重置。
9. 点击“停止自动识别”。

通过标准：

- 启动过程中显示“正在启动相机识别，请确认浏览器权限。”
- 失败时显示“自动识别启动失败，请检查相机权限或模型是否已加载。”
- 成功时状态会显示当前识别状态和置信度。
- 识别到已训练状态时，文字、图片或 3D 模型按屏幕锚点显示。
- 未识别到已训练状态或置信度不足时，虚拟物体不显示。
- 纯音频输出或附加音频只在识别到对应状态时触发。
- 识别灵敏度和状态计数适合现场调试使用。
- 停止后可再次启动。

### 4.6 App 内测试记录

1. 回到首页，找到“真机测试记录”。
2. 填写手机型号、系统版本和浏览器。
3. 勾选已通过的检查项：页面能打开、相机能开启、训练能完成、识别够流畅、PWA 可安装。
4. 在“问题备注”里写清楚异常现象，例如“训练成功，但识别偶尔慢半拍”。
5. 点击“保存记录”会把结果保存到当前手机浏览器。
6. 点击“复制记录”可以复制一段中文报告，方便发给开发者排查。

通过标准：

- 刷新页面后，上一次保存的记录仍能显示。
- 复制出的报告包含手机、系统、浏览器、地址、检查项和备注。
- 如果某项未勾选，报告中会显示“需排查”。

### 4.7 项目保存与再次编辑

1. 完成训练，并完成编辑页的文字、图片、3D 模型或音频绑定。
2. 点击“保存项目”。
3. 返回首页。
4. 在“本机项目”中找到刚保存的项目。
5. 点击“继续编辑”。
6. 进入测试页，点击“启动自动识别”。
7. 返回首页，尝试重命名该项目。
8. 如确认不再需要，尝试删除该项目。

通过标准：

- 能回到编辑页。
- 已保存的文字、图片、3D 模型或音频仍在。
- 已保存的位置、大小、旋转和识别灵敏度仍保持原值。
- 如果保存前已经训练成功，重新打开项目后不需要重新训练，也可以直接启动真实相机识别。
- 项目重命名后，首页本机项目列表会显示新名称。
- 删除项目前会有确认，删除后该项目不再出现在本机项目列表中。
- 说明：项目、素材、绑定和轻量识别模型快照都保存在当前手机浏览器本地；换手机、换浏览器或清除站点数据后，本机项目不会自动同步。

## 6. 常见问题处理

### 页面显示“需要 HTTPS 或 localhost”

说明当前地址不是安全上下文。手机必须使用 HTTPS 域名访问；局域网 `http://电脑IP:端口` 通常不能开启摄像头。

### 页面显示“当前浏览器未提供 getUserMedia”

说明浏览器没有暴露相机 API。优先换 Chrome、Edge、Safari 或系统最新浏览器，并检查浏览器相机权限。

### PWA 无法安装

先确认：

- 当前地址是 HTTPS。
- `manifest.webmanifest` 可访问。
- service worker 已生成。
- 浏览器没有处于无痕模式或企业限制环境。

### iOS / 鸿蒙没有空间 AR

第一版允许降级为相机叠加模式。核心目标是能打开相机、能识别状态、能叠加输出；空间锚定和深度感知留到后续原生封装或平台专属能力增强。
